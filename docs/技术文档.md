# 《九价守护计划》技术文档

## 一、项目结构

```
project/
├── index.html              # 主页面（含 9 个屏幕的 HTML 结构）
├── css/
│   └── style.css           # 全局样式表（响应式 + 动画 + 组件）
├── js/
│   └── main.js             # 游戏引擎（状态管理 + 交互逻辑）
├── assets/                 # 静态资源目录
├── docs/
│   ├── 作品申报文档.md
│   ├── 技术文档.md         # 本文档
│   └── 四分钟演示脚本.md
└── README.md
```

技术特点：
- **零依赖：** 纯原生 HTML/CSS/JS 实现，不依赖任何第三方库或框架
- **零外部 CDN：** 离线打开 index.html 即可运行，无需网络连接
- **移动优先：** 响应式布局，优先适配移动端，同时兼顾桌面端

## 二、HTML 结构说明

### 2.1 架构概览

采用单页面（SPA）架构，所有屏幕通过 Section 标签承载，通过 CSS `display` 属性控制显示/隐藏。

```
<div id="app">
  <section id="screen-home" class="screen active">     <!-- 首页 -->
  <section id="screen-mission1" class="screen">         <!-- 任务一 -->
  <section id="screen-mission2" class="screen">         <!-- 任务二 -->
  <section id="screen-mission3" class="screen">         <!-- 任务三 -->
  <section id="screen-mission4" class="screen">         <!-- 任务四 -->
  <section id="screen-mission5" class="screen">         <!-- 任务五 -->
  <section id="screen-mission6" class="screen">         <!-- 任务六 -->
  <section id="screen-ending" class="screen">           <!-- 结局页 -->
  <section id="screen-knowledge" class="screen">        <!-- 知识资料 -->
  <section id="screen-reference" class="screen">        <!-- 参考来源 -->
</div>
```

### 2.2 关键结构

#### 免责声明横幅（disclaimer-banner）
固定在页面顶部的全局横幅，包含科普免责声明和"详细说明"按钮。

#### 演示模式叠加层（demo-overlay）
固定在页面底部的半透明工具栏，显示演示进度和退出按钮。

#### 徽章获得通知（badge-toast）
浮动弹出通知，玩家完成任务获得徽章时短暂显示。

#### 模态框系统
- `modal-knowledge`：知识卡片详细弹窗
- `modal-disclaimer`：完整免责声明弹窗

### 2.3 各任务 HTML 结构模式

每个任务屏幕遵循统一结构：
```
<section id="screen-missionN">
  <div class="mission-header">    <!-- 返回按钮 + 进度条 + 任务编号 -->
  <div class="mission-body">      <!-- 任务标题 + 说明 + 交互区 + 反馈区 -->
  <div class="mission-footer">    <!-- 完成按钮 -->
</section>
```

## 三、CSS 响应式与动画说明

### 3.1 CSS 变量体系

使用 CSS 自定义属性定义全局设计令牌：
- 主色：`--blue`、`--green`、`--pink`、`--yellow`
- 语义色：`--bg`、`--bg-card`、`--text`、`--text-secondary`
- 视觉效果：`--shadow`、`--radius`、`--transition`
- 布局：`--max-width: 480px`（移动端最大宽度）

### 3.2 响应式断点

| 断点 | 目标设备 | 关键变化 |
|------|----------|----------|
| 默认 (< 768px) | 手机竖屏 | 单列布局，紧凑间距，触摸友好 |
| ≥ 768px | 平板 | 双列/三列，适中间距 |
| ≥ 1024px | 桌面 | 三列/四列，舒适间距，居中显示 |

### 3.3 CSS 动画清单

| 动画名称 | 用途 | 实现方式 |
|----------|------|----------|
| `fadeIn` | 屏幕切换入场 | opacity + translateY |
| `fadeInUp` | 卡片依次进入 | opacity + translateY（带延迟）|
| `pulse` | 正确操作反馈 | scale 缩放 |
| `bounce` | 首页图标浮动 | translateY 弹跳 |
| `glow` | 防护盾发光 | box-shadow 脉冲 |
| `shake` | 错误操作反馈 | translateX 抖动 |
| `float` | 背景粒子漂浮 | translateY + rotate |
| `shieldGlow` | 双重防线完成 | filter drop-shadow |
| `confetti` | 完成奖励 | translateY + rotate + opacity |
| `stampDown` | 盖章动画 | scale + rotate + opacity |
| `progressPulse` | 进度条流动 | background-position |
| `spinIn` | 徽章旋转入场 | rotate + scale |

### 3.4 移动端优化

- `-webkit-tap-highlight-color: transparent` 去除点击高亮
- `user-select: none` 防止误选文字
- 按钮最小触摸区域 ≥ 44×44px
- `-webkit-text-size-adjust: 100%` 防止字体缩放
- `viewport-fit=cover` 支持刘海屏

## 四、JavaScript 游戏状态管理

### 4.1 State 对象

```javascript
const state = {
  currentMission: 0,       // 当前所在屏幕（0=首页, 1-6=任务, 7=结局）
  score: 0,                // 累计得分（满分100）
  maxScore: 100,
  badges: [],              // 已获得徽章列表
  demoMode: false,         // 是否处于演示模式
  missionsCompleted: [...], // 各任务完成状态
  m1: {...},               // 任务一临时状态
  m2: {...},               // 任务二临时状态
  // ... 各任务独立状态
};
```

### 4.2 核心函数

| 函数 | 功能 |
|------|------|
| `initApp()` | 应用入口，初始化首页 |
| `startGame()` | 重置状态，开始正常游戏 |
| `startDemo()` | 重置状态，进入演示模式 |
| `goToMission(n)` | 跳转到第 n 个任务 |
| `initMissionN()` | 初始化第 n 个任务的交互元素 |
| `completeMission(n)` | 完成第 n 个任务，发放徽章，加分 |
| `showResult()` | 显示结局页，计算评分和称号 |
| `resetGame()` | 重置所有状态到初始值 |
| `updateProgress(n)` | 更新进度条宽度 |

### 4.3 任务一交互逻辑

- 卡片点击 → `handleMission1Click(el, card)`
- 正确卡片：绿色高亮 + 加分 + 科普事实弹出
- 错误卡片：红色抖动 + 温和提示
- 累计 3 次正确自动完成

### 4.4 任务二交互逻辑

- 支持**拖拽**（desktop `dragstart/drop`）和**点击选择**（mobile 友好）
- `placeType(typeNum, zoneRisk, dropZone)`：验证分类正确性
- 正确：绿色光效 + 卡片移动到目标区
- 错误：抖动 + 解释 + 1.5 秒后恢复

### 4.5 任务三交互逻辑

- 30 秒倒计时匹配游戏
- `selectAntigen(el)`：选中左侧抗原线索
- `matchWithCell(cellEl)`：匹配到右侧免疫细胞
- 计时器独立管理，可清除重来

### 4.6 任务四交互逻辑

- 流程卡片序列排序
- `selectStep(el, step)`：验证当前步骤是否为正确顺序
- 正确：加入序列显示区，编号递增
- 完成所有 7 步触发盖章动画

### 4.7 任务五交互逻辑

- 双按钮切换状态
- `checkShieldStatus()`：检测双重防线状态
- 单防线激活 → 提示文字 + 知识点展示
- 双防线激活 → 盾牌合成动画 + 完成任务

### 4.8 任务六交互逻辑

- `answerQuiz(item, userAnswer, quiz)`：验证答案
- 正确/错误：对应样式 + 解释展示 + 禁用选项
- 6 题全部答完自动完成

### 4.9 评分系统

- 每个正确操作 +2~5 分
- 完成任务 +5 分
- 全部 6 个任务额外 +10 分
- 满分 100 分
- 称号分级：≥90 首席 / ≥70 守护员 / ≥50 学习者 / <50 探索者

## 五、演示模式实现说明

### 5.1 状态标志

通过 `state.demoMode` 全局标志控制。设置为 `true` 后：
1. 底部叠加层（demo-overlay）显示
2. 进度计数（0/6）实时更新
3. 每关自动高亮正确操作
4. 显示口播提示文案

### 5.2 自动引导

任务五（双重防线）在演示模式下自动延时点亮：
```javascript
if (state.demoMode) {
  setTimeout(() => btnVaccine.click(), 800);
  setTimeout(() => btnScreen.click(), 1800);
}
```

### 5.3 高亮机制

CSS 类 `demo-highlight` 应用黄色脉冲边框高亮，配合 `.demo-hint-text` 显示口播文字。

### 5.4 退出演示

点击"退出演示"按钮 → 重置游戏状态 → 回到首页。

### 6.2 移动端兼容

- **iOS Safari：** 支持 viewport-fit=cover 刘海屏适配
- **Android Chrome：** 完全兼容

- **触摸事件：** 使用 click 事件（移动端天然支持）+ drag 事件（桌面端）

### 6.3 渐进增强

- 不支持 Grid 的旧浏览器回退到 Flexbox
- 不支持 CSS 变量的旧浏览器使用硬编码颜色
- 禁止动画的用户通过 `prefers-reduced-motion` 媒体查询获得静态体验
